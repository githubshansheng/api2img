## 2. 引言

### 2.1 编写目的

本文档定义 AI 图片生成与编辑工作台的系统边界、功能规格、数据规格、接口规则、异常处理、非功能要求与验收口径。研发应以本文档作为需求拆解和技术设计输入；测试应以本文档作为测试用例、验收清单和缺陷判定依据。

### 2.2 产品范围

系统面向内容创作者、电商运营、设计师、开发者和企业客户，提供多模型 AI 图片生成、参考图编辑、批量生成、模型对比、历史记录、素材模板、图片识别、推理测试、API Key 管理、对接代码生成、存储配置和数据埋点能力。

首期交付建议以生成图片页、批量抽卡、模型能力配置、费用展示、cURL 对接代码、API Key 管理、错误处理和基础下载为核心闭环；历史记录、素材模板、图片识别和模型对比按 P1 逐步完善；推理测试和高级存储按 P2 或灰度策略推进。

## 3. 总体描述

### 3.1 产品视角

系统是一个浏览器端 AI 图片工作台，可连接第三方图片生成与视觉理解模型。系统应支持前端直连、后端代理或混合模式，但接口、安全和计费边界必须在项目启动阶段确认。

系统由以下核心能力组成：

1. 模型能力配置层：维护模型、端点、价格、尺寸、分辨率、质量、参考图上限、临时限制和公告。
2. 创作工作台层：提供提示词输入、参考图上传、参数配置、生成请求、结果预览、下载和历史记录。
3. 生产效率层：提供批量生成、提示词模板、素材模板、模型对比和复用能力。
4. 开发者支持层：提供 API Key、端点说明、cURL 请求示例和调用日志入口。
5. 存储与同步层：提供本地历史、云端同步、自定义存储、本地文件夹归档和临时链接处理。
6. 观测与治理层：提供埋点、错误映射、安全脱敏、合规提示和版本更新提示。

### 3.2 用户角色

| 角色 | 权限/能力 | 关键目标 |
| --- | --- | --- |
| 未登录用户 | 本地 API Key、本地历史、基础生成、基础下载 | 快速试用生成能力 |
| 登录用户 | 云同步、素材模板同步、跨设备历史、更多账号能力 | 持续创作与复用 |
| 内容创作者 | 模板、优化、生成、下载、再生成 | 快速产出内容素材 |
| 电商运营 | 参考图、商品模板、图片识别、批量生成 | 生产商品图和文案 |
| 设计师 | 模型对比、画幅控制、质量控制 | 对比效果并选择模型 |
| 开发者 | API Key、cURL、端点文档、日志 | 验证模型并接入 API |
| 企业客户 | 稳定性、价格、企业分组、服务支持 | 评估生产可用性 |

### 3.3 运行环境

| 环境项 | 要求 |
| --- | --- |
| 浏览器 | 支持现代 Chromium、Safari、Firefox；本地文件夹归档仅 Chromium 且安全上下文可用 |
| 网络 | 能访问本系统、第三方模型 API、对象存储和认证服务 |
| 存储 | localStorage、IndexedDB、默认云端存储、自定义 R2/OSS 可按配置启用 |
| 认证 | 首期可支持 Google 登录；是否扩展其它登录方式待确认 |
| 文件能力 | 支持 JPG、PNG 上传；模型对比可按能力支持 GIF |
| 安全上下文 | 本地目录授权、剪贴板和文件写入能力需 HTTPS 或 localhost |

### 3.4 约束

1. 模型能力差异必须由配置驱动，不能假设所有模型支持同一套参数。
2. API Key 默认不得明文显示，复制代码默认使用占位密钥。
3. 图片和提示词可能包含隐私信息，系统必须在提交前或设置中明确第三方发送边界。
4. 本地保存的 Key 若仅 Base64 或等价弱编码，不得在产品文案或安全设计中宣称强加密。
5. 上游临时限制、官方限流和模型下线必须可配置恢复，避免每次发版。
6. 短期有效图片链接必须提示用户及时保存或归档。
7. 内容合规策略在首期必须至少提供风险提示和错误映射；是否做前置审核待确认。

### 3.5 假设与依赖

1. 第三方模型 API 可按用户 API Key 或系统代理 Key 发起请求。
2. PRD 中列出的模型、价格、速度和能力来自 2026-07-06 的线上可见信息，后续以配置为准。
3. 提示词模板首期可以通过 JSON 静态文件或配置服务加载。
4. 云同步能力依赖登录态和后端数据服务。
5. 自动归档依赖浏览器 File System Access API，不支持时必须降级。

## 4. 功能需求

### 4.1 全局框架

| 编号 | 优先级 | 需求规格 | 验收标准 |
| --- | --- | --- | --- |
| SRS-FR-GLOB-001 | P0 | 系统应展示左侧工作台导航，包含生成图片、批量生成、模型对比、历史记录、素材模板、识别图片、推理测试。 | 导航项可见；点击后进入对应页面或展示未开放状态。 |
| SRS-FR-GLOB-002 | P0 | 系统应展示顶部公告条，用于模型状态、尺寸限制、模板升级、临时规则等通知。 | 公告可轮播；支持手动切换；当前模型存在限制时优先展示限制说明。 |
| SRS-FR-GLOB-003 | P1 | 系统应提供语言切换，至少支持简体中文、English、繁体中文、俄语。 | 切换语言后核心菜单、按钮、提示、模板分类和错误文案同步更新。 |
| SRS-FR-GLOB-004 | P1 | 系统应提供版本更新提示，可选择立即更新或稍后更新。 | 新版本可触发提示；稍后更新不阻塞当前操作。 |
| SRS-FR-GLOB-005 | P1 | 系统应提供登录入口、设置入口、日志入口、充值入口、文档中心和企业服务入口。 | 入口可点击；外链打开时不破坏当前工作台状态。 |

### 4.2 模型选择与能力驱动

| 编号 | 优先级 | 需求规格 | 验收标准 |
| --- | --- | --- | --- |
| SRS-FR-MODEL-001 | P0 | 系统应通过模型元数据渲染模型列表，不得在页面逻辑中硬编码参数能力。 | 更新模型配置后，模型名称、价格、端点、参数和限制可同步变化。 |
| SRS-FR-MODEL-002 | P0 | 模型列表首期应覆盖 PRD 中识别的 16 个线上模型。 | 16 个模型均可在模型选择器中被识别，至少展示名称、速度、价格和能力摘要。 |
| SRS-FR-MODEL-003 | P0 | 切换模型后，系统应同步刷新尺寸、分辨率、质量、数量、参考图上限、费用区、结果说明和 cURL。 | 任一模型切换后，不出现与该模型不兼容但仍可提交的参数。 |
| SRS-FR-MODEL-004 | P0 | 系统应支持模型临时限制配置，如 size 参数失效、高分辨率禁用、官方限流提示。 | 限制打开后，受影响参数置灰或固定，公告和参数区均提示原因。 |
| SRS-FR-MODEL-005 | P1 | 系统应支持模型下线回退到默认模型。 | 本地存储模型不存在时，页面自动选择默认模型并提示或记录日志。 |
| SRS-FR-MODEL-006 | P1 | 系统应支持旧模型名或 preview 端点兼容开关。 | 开关启用后，请求端点和 cURL 同步切换；关闭后回到正式端点。 |

#### 4.3.6 生成流程

| 编号 | 优先级 | 需求规格 | 验收标准 |
| --- | --- | --- | --- |
| SRS-FR-RUN-001 | P0 | 用户点击开始生成后，系统应进行前置校验。 | 校验提示词、参考图、API Key、余额、模型能力、文件格式和文件大小。 |
| SRS-FR-RUN-002 | P0 | 请求发送后，系统应进入生成中状态并禁用重复提交。 | 生成中按钮不可重复点击；页面展示当前模型和进度/等待状态。 |
| SRS-FR-RUN-003 | P0 | 生成成功后，系统应在结果区展示图片。 | 图片可见，支持后续操作。 |
| SRS-FR-RUN-004 | P0 | 生成失败后，系统应展示错误摘要、错误码、重试入口和排查建议。 | 错误信息不包含完整密钥；可展开详细错误。 |
| SRS-FR-RUN-005 | P1 | 生成结果应写入历史记录。 | 成功、失败、部分成功均有对应记录或状态。 |

| --- | --- | --- | --- |

| SRS-FR-RESULT-006 | P1 | 支持本地文件夹自动归档。 | 支持浏览器可授权目录并自动保存；不支持时隐藏或降级。 |

## 5. 外部接口需求

### 5.1 用户界面接口

1. 所有主要操作按钮应在执行中提供禁用态、加载态或进度态。
2. 表单校验应就近提示，避免只在全局 Toast 中展示。
3. 参数因模型能力不可用时，应以禁用态和原因说明呈现。
4. 错误详情应允许展开查看，但默认不暴露敏感密钥。
5. 结果图片应支持预览大图；若浏览器或跨域限制导致无法下载，应提供降级保存方式。

## 7. 校验与异常处理

### 7.1 前置校验

| 场景 | 系统行为 |
| --- | --- |
| 提示词为空且无参考图 | 阻止提交，提示输入提示词或上传参考图 |
| API Key 缺失 | 引导打开设置并输入 API Key |
| 余额不足 | 阻止提交并引导充值 |
| 文件格式不支持 | 阻止上传并提示支持格式 |
| 文件超过 20 MB | 阻止上传并提示文件名和限制 |
| 参考图数量超限 | 阻止继续添加并提示上限 |
| 模型参数不兼容 | 禁用不可用参数并展示说明 |
| 自定义存储配置无效 | 连接测试失败，不保存或提示回退默认存储 |

### 7.2 上游错误映射

| 场景 | 用户可见标题 | 处理要求 |
| --- | --- | --- |
| HTTP 400 | 请求参数错误 | 提示检查提示词、参数或敏感内容 |
| HTTP 401 | API Key 认证失败 | 提示检查 Key 是否正确或过期 |
| HTTP 403 | 访问权限不足 | 提示权限或余额问题 |
| HTTP 429 | 出图失败异常 | 对 `gpt-image-2` 提示可能需要 `image2Enterprise` 企业分组 |
| HTTP 500 | 服务器内部错误 | 提示稍后重试 |
| HTTP 502/503/504 | 服务暂时不可用 | 提示稍后重试 |
| Google candidatesTokenCount 为 0 | 谷歌拒绝出图 | 提示修改提示词或参考图 |
| finishReason=PROHIBITED_CONTENT | 违禁内容 | 提示内容政策风险 |
| finishReason=SAFETY | 安全过滤 | 提示调整提示词 |
| finishReason=NO_IMAGE | 未生成图片 | 提示明确图片生成或编辑意图 |
| finishReason=RECITATION | 引用限制 | 提示避免受保护内容 |
| finishReason=MAX_TOKENS | Token 超限 | 提示缩短输入或降低复杂度 |
| Failed to fetch / 网络中断 | 连接中断，本次可能未扣费 | 提示重试、切换网络、减少参考图 |
| 临时图片链接失效 | 图片链接已失效 | 提示重新生成、查看归档或及时保存 |

### 7.3 错误详情展示

1. 默认展示用户可理解的标题、摘要和建议。
2. 可展开技术详情，但不得展示完整 API Key。
3. 原始响应最多默认展示摘要；完整日志是否保存取决于安全策略。
4. 对可能未扣费的网络中断，应明确与上游失败扣费风险区分。
5. 重试按钮应只重试当前失败任务，不重复提交成功任务。

| SRS-NFR-UX-001 | 所有阻塞性失败都应给出下一步行动。 | 错误提示包含重试、修改输入、充值、设置 Key 或查看文档等路径。 |
| SRS-NFR-UX-002 | 不支持的浏览器能力应隐藏或降级。 | Safari/Firefox 不展示不可用的本地目录自动归档入口，或明确说明不可用。 |
| SRS-NFR-UX-003 | 模板、素材、历史等复用动作应保留用户已有输入的处理策略。 | 追加、替换或覆盖前有明确规则。 |
| SRS-NFR-UX-004 | 长耗时生成应持续反馈状态。 | 展示处理中、排队、重试中或阶段进度。 |

### 8.4 兼容性

| 编号 | 需求规格 | 验收标准 |
| --- | --- | --- |
| SRS-NFR-COMP-001 | 系统应兼容主流桌面浏览器。 | Chrome、Edge、Safari、Firefox 可完成基础生成流程。 |
| SRS-NFR-COMP-002 | 本地归档仅在支持 File System Access API 的浏览器启用。 | 非支持浏览器不报错。 |
| SRS-NFR-COMP-003 | 模型端点差异应通过适配层处理。 | 新增模型不需要改动页面主流程。 |

### 8.5 可维护性

| 编号 | 需求规格 | 验收标准 |
| --- | --- | --- |
| SRS-NFR-MAINT-001 | 模型配置、价格、限制、公告应配置化。 | 运维或配置变更可调整，不依赖代码发版。 |
| SRS-NFR-MAINT-002 | 提示词模板应支持独立数据文件或配置服务。 | 模板变更不影响生成主流程。 |

| 编号 | 需求规格 | 验收标准 |

| SRS-NFR-COMP-PRIV-002 | 系统不得扩散违规内容细节。 | 内容违规时只提示风险和调整建议。 |
